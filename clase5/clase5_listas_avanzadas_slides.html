🔄 ¿Por qué Doblemente Enlazadas?

Solucionando las limitaciones

⚠️ Problema con Lista Simple

❌ Solo navegación hacia adelante

❌ Eliminar nodo requiere el anterior

❌ No hay acceso eficiente al final

// Para eliminar último nodo
while(actual.siguiente.siguiente != null) {
    actual = actual.siguiente; // O(n)
}

✅ Solución: Lista Doble

✅ Navegación bidireccional

✅ Eliminación más eficiente

✅ Acceso rápido desde ambos extremos

// Con referencia al nodo anterior
nodoActual.prev.next = nodoActual.next;
nodoActual.next.prev = nodoActual.prev; // O(1)

💡 Idea clave: Cada nodo conoce a su vecino anterior

📦 DATO + 🔗 NEXT + ⬅️ PREV

public class DoublyNode<T> {
    T data;
    DoublyNode<T> next; // Referencia al siguiente
    DoublyNode<T> prev; // ¡Nueva referencia al anterior!

    public DoublyNode(T data) {
        this.data = data;
        this.next = null;
        this.prev = null;
    }
}

public class DoublyLinkedList<T> {
    DoublyNode<T> head;
    DoublyNode<T> tail; // ¡Referencia al último también!
    int size;
}

🔑 Ventajas clave:

🏃‍♂️ Navegación bidireccional

Puedo ir hacia adelante y hacia atrás

🎯 Eliminación eficiente

Con referencia al nodo, eliminación O(1)

📍 Acceso a ambos extremos

HEAD para el inicio, TAIL para el final

🔧 Operaciones más flexibles

Inserción y eliminación en cualquier punto

🎯 ¿Cuándo usar cada una?

Guía de decisión

🔗 USA SIMPLE CUANDO:

  • ➡️ Solo navegación hacia adelante
    Recorridos secuenciales únicos
  • 💾 Memoria limitada
    Cada byte cuenta
  • ⚡ Operaciones simples
    Insert/delete principalmente al inicio
  • 🏗️ Implementación básica
    Stack, Queue simple
  • 📚 Aprendizaje
    Entender el concepto fundamental

🔄 USA DOBLE CUANDO:

  • ⬅️➡️ Navegación bidireccional
    Necesito ir y venir frecuentemente
  • 🗑️ Eliminaciones frecuentes
    Con referencia directa al nodo
  • 📝 Editor de texto
    Cursor que se mueve en ambas direcciones
  • 🌐 Navegador web
    Historial con back/forward
  • 🎵 Playlist música
    Anterior/siguiente con facilidad

🏆 Casos de uso específicos:

🔗 Lista Simple perfecta para:

  • • Stack (LIFO) - solo importa el tope
  • • Cola simple (FIFO) básica
  • • Lista de tareas pendientes
  • • Procesamiento secuencial de datos

🔄 Lista Doble perfecta para:

  • • Deque (Double-ended queue)
  • • Cache con LRU (Least Recently Used)
  • • Undo/Redo en aplicaciones
  • • Navegación en interfaces de usuario

🔁 Listas Circulares

Cerrando el círculo

💡 Idea: El último nodo apunta al primero

No hay "NULL" al final - es un círculo infinito

Lista Circular Simple

"A"
next →
→
"B"
next →
→
"C"
next →
⤴
C.next apunta de vuelta a A
public void add(T data) {
    Node<T> newNode = new Node<>(data);
    if(head == null) {
        head = newNode;
        newNode.next = head; // ¡Apunta a sí mismo!
    } else {
        // Encontrar el último nodo
        Node current = head;
        while(current.next != head) {
            current = current.next;
        }
        current.next = newNode;
        newNode.next = head; // ¡Cierra el círculo!
    }
}

🎯 Aplicaciones Listas Circulares

🎮 Round Robin

// Jugadores: Ana → Bob → Carlos → Ana
current = current.next; // Siguiente turno

Perfecto para turnos, CPU scheduling

🎵 Playlist Música

while(true) {
    reproducir(current.cancion);
    current = current.next; // Infinito
}

Repetir automáticamente

⚠️ CUIDADO: Loops Infinitos

// ❌ PELIGRO - Loop infinito
Node current = head;
while(current != null) { // ¡Nunca será null!
    current = current.next;
}

// ✅ CORRECTO - Detectar ciclo
Node current = head;
do {
    // procesar nodo
    current = current.next;
} while(current != head);
