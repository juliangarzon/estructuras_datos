🧩 Concepto de Nodo

La pieza fundamental

¿Qué es un Nodo?

📦 DATO + 🔗 REFERENCIA
class Nodo<T> {
    T dato; // El valor que guardamos
    Nodo<T> siguiente; // Referencia al próximo nodo

    public Nodo(T dato) {
        this.dato = dato;
        this.siguiente = null;
    }

    public String toString() {
        return dato + " → " +
               (siguiente == null ? "null" : "[...]");
    }
}

🔑 Diferencia Fundamental:

ArrayList
Memoria
CONTIGUA
VS
Nodos
Memoria
DISPERSA

ArrayList: [A][B][C][D] en memoria consecutiva
Nodos: A→B→C→D conectados por referencias

🔗 Listas Enlazadas - Concepto

Secuencia de nodos conectados

Anatomía de una Lista Enlazada

HEAD
↓
"A"
next →
→
"B"
next →
→
"C"
next →
→
NULL
// Creando la lista manualmente
Nodo<String> primero = new Nodo<>("A");
Nodo<String> segundo = new Nodo<>("B");
Nodo<String> tercero = new Nodo<>("C");

// Conectando los nodos
primero.siguiente = segundo;
segundo.siguiente = tercero;
tercero.siguiente = null; // Último nodo

// HEAD apunta al primer nodo
Nodo<String> head = primero;

📋 Componentes Clave:

  • 🎯 HEAD: Referencia al primer nodo
  • 🧩 NODOS: Cada uno contiene dato + referencia
  • 🏁 NULL: El último nodo apunta a null
  • 🔗 ENLACES: Conexiones que forman la secuencia

✅ Ventajas vs ArrayList

¿Por qué necesitamos Listas Enlazadas?

📊 ArrayList

Insertar al inicio:

O(n)
Mueve TODOS los elementos
  • ❌ Inserción costosa al inicio
  • ❌ Eliminación costosa al inicio
  • ❌ Redimensionar cuando se llena
  • ❌ Memoria desperdiciada
VS

🔗 LinkedList

Insertar al inicio:

O(1)
Solo cambiar referencias
  • ✅ Inserción rápida al inicio
  • ✅ Eliminación rápida al inicio
  • ✅ Sin redimensionar
  • ✅ Memoria dinámica
// ArrayList: insertar al inicio
list.add(0, "nuevo"); // O(n) - mueve todo

// LinkedList: insertar al inicio
nuevoNodo.siguiente = head; // O(1) - solo referencias
head = nuevoNodo;

❌ Desventajas vs ArrayList

No todo es perfecto

Operación ArrayList LinkedList ¿Por qué?
get(índice) O(1) O(n) ArrayList: acceso directo. LinkedList: recorrer hasta el índice
add al inicio O(n) O(1) ArrayList: mover elementos. LinkedList: cambiar referencias
add al final O(1)* O(n) ArrayList: posición conocida. LinkedList: recorrer hasta el final
Memoria por elemento Solo dato Dato + referencia LinkedList necesita memoria extra para punteros

🎯 Problemas específicos de LinkedList:

🚫 No Acceso Aleatorio

// ArrayList
String elemento = lista.get(100); // O(1)

// LinkedList
// Debe recorrer 100 nodos
Nodo actual = head;
for(int i = 0; i < 100; i++) {
    actual = actual.siguiente; // O(n)
}

💾 Más Memoria

ArrayList:
[Dato][Dato][Dato]

LinkedList:
[Dato|Ptr→][Dato|Ptr→][Dato|Ptr→]
~33% más memoria

🎯 ¿Cuándo usar cada una?

Guía de decisión

📊 USA ARRAYLIST CUANDO:

  • 🎯 Acceso frecuente por índice
    lista.get(50)
  • 🔍 Búsquedas aleatorias
    Necesitas elementos específicos
  • 📈 Más inserciones al final
    Crecimiento por el final
  • 💾 Memoria limitada
    Cada byte cuenta
  • 📊 Cálculos matemáticos
    Procesamiento de arrays

🔗 USA LINKEDLIST CUANDO:

  • ⚡ Muchas inserciones al inicio
    Historial, Stack, Queue
  • 🗑️ Muchas eliminaciones al inicio
    Procesamiento de colas
  • 🔄 Inserciones en el medio
    Con referencia al nodo
  • 📏 Tamaño muy variable
    No sabes cuántos elementos
  • 🚫 Poco acceso aleatorio
    Solo recorridos secuenciales

🏆 Casos de uso reales:

📊 ArrayList perfecto para:

  • • Lista de estudiantes (acceso por ID)
  • • Catálogo de productos (búsquedas)
  • • Datos para gráficos
  • • Matrices y cálculos

🔗 LinkedList perfecto para:

  • • Historial del navegador
  • • Lista de reproducción (insertar canciones)
  • • Sistema de deshacer/rehacer
  • • Cola de impresión

// ¿Cómo eliminar el ÚLTIMO elemento eficientemente?
public void removeLast() {
    // Debo recorrer TODA la lista para llegar al penúltimo
    Nodo actual = head;
    while(actual.siguiente.siguiente != null) {
        actual = actual.siguiente; // O(n) - ¡Lento!
    }
    actual.siguiente = null;
}
